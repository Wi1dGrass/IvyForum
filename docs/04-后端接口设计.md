# 04 后端接口设计

> 所有接口前缀 `/api`。统一响应体 `{ code, message, data }`，分页 `data` 为 `{ records, total, page, size }`。
> 需登录接口携带请求头 `Authorization: Bearer <token>`。

## 1. 错误码段
- 1xxx 认证 / 授权
- 2xxx 参数校验
- 3xxx 业务
- 5xxx 服务端

## 2. 接口分组

### 2.1 认证 AuthController (`/api/auth`)

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| POST | /register | 游客 | 注册 |
| POST | /login | 游客 | 登录，返回 JWT |
| POST | /logout | 登录 | 登出 |
| GET  | /me | 登录 | 获取当前用户信息 |
| PUT  | /password | 登录 | 修改密码 |

**POST /api/auth/register**
```json
// req
{ "username": "stu01", "password": "123456", "nickname": "小明", "email": "a@b.com" }
// resp data
{ "userId": 1, "username": "stu01", "role": "STUDENT" }
```

**POST /api/auth/login**
```json
// req
{ "username": "stu01", "password": "123456" }
// resp data
{ "token": "eyJ...", "userId": 1, "username": "stu01", "role": "STUDENT", "nickname": "小明" }
```

### 2.4 帖子 PostController

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| GET  | /api/posts | 全部 | 分页；参数 channelId, tagIds, keyword, sort=latest\|hot, page, size |
| GET  | /api/posts/hot | 全部 | 热门榜 |
| GET  | /api/posts/{id} | 全部 | 浏览量 +1 |
| POST | /api/posts | 学生/教师 | 发帖 |
| PUT  | /api/posts/{id} | 作者/管理员 | 编辑 |
| DELETE | /api/posts/{id} | 作者/管理员 | 逻辑删除 |
| POST | /api/admin/posts/{id}/top | 管理员 | 切换置顶 |
| POST | /api/admin/posts/{id}/essence | 管理员 | 切换加精 |

**POST /api/posts req**
```json
{
  "channelId": 1,
  "title": "Java 安装问题",
  "content": "# 安装\n...",
  "tagIds": [1, 2]
}
```

**GET /api/posts 列表项 vo**
```json
{
  "postId": 1, "authorId": 1, "authorNickname": "小明",
  "authorAvatar": null,
  "channelId": 1, "channelName": "学习交流",
  "title": "Java 安装问题",
  "viewCount": 10, "likeCount": 3, "commentCount": 2, "collectCount": 0,
  "isTop": 0, "isEssence": 0,
  "tags": [{"tagId":1,"name":"Java","color":"#409EFF"}],
  "createTime": "2026-06-26 10:00:00"
}
```

### 2.5 评论 CommentController
