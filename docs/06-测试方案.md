# 06 测试方案

## 1. 测试职责
- 组员 B 负责：接口测试用例编写、Postman 集合维护、接口回归
- 组员 A（后端）：JUnit 单元测试、Controller 冒烟测试
- 组员 B（前端）：关键页面操作核对清单（手动）

## 2. 后端单元测试

### 工具
- JUnit 5
- Mockito
- Spring Boot Test（@SpringBootTest）
- MockMvc（Controller 冒烟）

### 测试范围与覆盖率目标
| 层 | 范围 | 目标 |
|----|------|------|
| Service | 帖子发布、评论树构建、点赞去重、热门分计算、举报状态流转 | ≥60% |
| Controller | 5-10 个关键接口冒烟 | 路径覆盖 |
| Mapper | 复杂查询 | 不强制 |

### 测试源码位置
```
backend/src/test/java/com/school/forum/
├── service/
│   ├── PostServiceTest.java
│   ├── CommentServiceTest.java
│   ├── LikeServiceTest.java
│   ├── HotPostServiceTest.java
│   └── ReportServiceTest.java
└── controller/
    └── PostControllerTest.java
```

### 示例测试用例（PostService）
| 用例 | 输入 | 预期 |
|------|------|------|
| 正常发帖 | 有效 title/channel/tagIds | status=NORMAL，评论计数 0 |
| 频道被禁用 | 已禁用频道 | 抛 BusinessException |
| 标签超限 | tagIds = 6 个 | 抛参数异常 |

## 3. 接口测试（Postman）

### 集合结构
```
SchoolForum
├── Auth          (register/login/logout/me)
├── Channel       (tree/crud)
├── Tag           (list/hot/crud)
├── Post          (list/hot/detail/create/update/delete/top/essence)
├── Comment       (list/create/delete)
├── Like          (toggle/status)
├── Collect       (toggle/list)
├── Notification  (list/unread-count/read)
├── Report        (create/admin list/handle)
└── User/Admin    (profile/posts/collects/ban/unban)
```

### 环境变量
- `{{baseURL}}` = http://localhost:8080/api
- `{{token}}` = 登录响应中获取（脚本自动写入）

### 进度产出
- 导出文件：`docs/postman/SchoolForum.postman_collection.json`
- 环境文件：`docs/postman/Schoolforum.postman_environment.json`

### 用例表（Excel）
列：用例ID / 模块 / 接口 / 方法 / 前置条件 / 输入 / 预期响应 / 实际结果 / 通过

样例：

| ID | 模块 | 接口 | 方法 | 前置 | 输入 | 预期 | 结果 |
|----|------|------|------|------|------|------|------|
| AUTH-001 | Auth | /auth/login | POST | 已注册 | 正确账号密码 | code=0，data.token 非空 | 通过 |
| AUTH-002 | Auth | /auth/login | POST | — | 错误密码 | code=1003 | 通过 |
| POST-001 | Post | /posts | GET | — | page=1&size=20 | code=0，records 长度 ≤20 | 通过 |
| POST-002 | Post | /posts | POST | 已登录 | 有效数据 | code=0，postId>0 | 通过 |
| POST-003 | Post | /posts | POST | 未登录 | — | code=1001 | 通过 |

## 4. 手动测试核对清单（前端关键流程）

- [ ] 游客可浏览首页 / 频道 / 详情
- [ ] 注册 → 登录 → 进入个人主页
- [ ] 发帖 → 列表出现 → 详情可看 → 编辑 → 删除消失
- [ ] 评论 + 楼中楼回复可见
- [ ] 点赞 / 收藏 可切换状态
- [ ] 通知：被评论后通知列表出现新消息
- [ ] 举报：管理员后台可看到待处理记录
- [ ] 管理员禁用频道 → 列表过滤
- [ ] 热门榜：定时刷新后排序变化

## 5. 测试环境
- 后端测试配置 `application-test.yml`
- 测试库 `school_forum_test`（独立，避免污染开发库）
- 测试用 H2 可选，但本项目直接连 MySQL test 库以保证一致性
