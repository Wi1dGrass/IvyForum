# 01 需求分析

## 1. 项目背景

本项目为计算机专业工程实践分组大作业，采用 Web 开发模式，前后端分离 + 测试。
- 组长：后端开发
- 组员 A：前端开发
- 组员 B：测试

## 2. 项目定位

**学校论坛** 面向校园师生提供内容社区服务：
- 学生可发帖求助、分享心得、反馈问题
- 教师可参与讨论、答疑、置顶官方答复
- 管理员维护论坛秩序

帖子按 **频道（Channel）** 分类，并贴 **标签（Tag）**，支持多维检索。

## 3. 角色与权限（RBAC）

| 角色 | 权限 |
|------|------|
| 游客 | 浏览公开帖子、按频道/标签/关键词搜索、查看用户主页 |
| 学生 | 游客权限 + 发帖/编辑自己帖子、评论/回复、点赞收藏、举报、收通知、编辑个人资料 |
| 教师 | 学生权限 + 帖子置顶权重、官方答复标识 |
| 管理员 | 全部权限 + 频道/标签维护、举报审核、封禁用户、屏蔽帖子 |

## 4. 功能模块清单

### M1 账户与权限
- 注册 / 登录（账号 + 密码）
- JWT 鉴权
- 角色分配
- 登出
- 密码修改
- 用户封禁 / 解封

### M2 频道与标签
- 频道（树形多级）CRUD
- 标签 CRUD
- 发帖时选择频道 + 多标签（最多 5 个）

### M3 帖子核心
- 发帖 / 编辑 / 删除（逻辑删除）
- Markdown 正文
- 浏览量统计
- 置顶、加精

### M4 互动
- 评论（一级评论）
- 楼中楼回复（自关联 parent_id）
- 点赞（帖子 + 评论）
- 收藏

### M5 检索
- 关键词搜索（标题 + 正文）
- 频道过滤
- 标签过滤
- 排序（最新 / 最热）

### M6 通知系统
- 被评论 / 回复 / @提及 / 审核结果 → 站内消息
- 未读数
- 已读标记

### M7 举报与审核
- 举报帖 / 评论（选择类型 + 备注）
- 管理员后台审核通过 / 驳回 / 删除

### M8 个人主页
- 用户资料（头像 / 签名 / 角色徽章）
- TA 的发帖 / 收藏 / 点赞历史

### M9 热门推荐
- 热度 = 浏览量 ×1 + 点赞 ×3 + 评论 ×2 + 时间衰减
- 定时任务刷新热门榜

### M10 后台管理
- 用户 / 频道 / 标签 / 举报 / 帖子综合管理面板

## 5. 非功能需求

### 性能
- 列表页分页 20 条 / 页
- 热门榜缓存

### 安全
- JWT 鉴权
- 密码 BCrypt 加密
- SQL 注入防护（MyBatis 参数化）
- XSS 过滤
- 接口限流（可选）

### 可维护
- SLF4J + Logback 日志
- Swagger / Knife4j 接口文档
- 统一异常处理、统一响应体
- 参数校验 Hibernate Validator
