## 3. 设计 Token（视觉体系）

- **主色**：`#4361ee`（校园蓝） + 渐变 `linear-gradient(135deg, #4361ee, #6c8aff)`
- **强调色**：`#f77f00`（活力橙）
- **背景**：`#f4f7fb`
- **卡片**：`#ffffff`，圆角 12px，柔和阴影
- **圆角**：8-12px（卡片）、999px（标签/按钮）
- **字体栈**：系统原生无衬线（PingFang SC / Microsoft YaHei）
- **布局**：顶栏 64px 粘性 + 左侧频道侧栏 240px + 内容区自适应
- **顶栏**：毛玻璃效果（`backdrop-filter: blur(10px)`）
- **过渡**：hover `translateY(-2px) + box-shadow` 微浮起

## 4. Mock 数据层

当 `VITE_USE_MOCK=true`（开发默认）时，Axios 使用自定义适配器 `mockAdapter` 拦截 `/api/*` 请求：
- 请求 → `handlers.ts` 根据 `method + path + query/body` 路由分发
- 内存数据池（`data.ts`）：4 用户、9 频道、7 帖子（含 Markdown 正文）、4 评论、2 通知、1 举报
- 模拟 JWT 鉴权：以 `Authorization: Bearer mock-user-{id}` 标识登录用户
- 保留完整 CRUD 状态变更（删除、编辑、点赞、收藏等）

## 5. 路由守卫

| meta | 行为 |
|------|------|
| `requiresAuth: true` | 未登录跳 `/login?redirect=xxx` |
| `roles: ['ADMIN']` | 非管理员跳首页 |

## 6. 与后端联调说明

1. 在 `frontend/.env.development` 中设置 `VITE_USE_MOCK=false` 和 `VITE_API_TARGET=http://localhost:8080`
2. 后端接口必须与 `docs/04-后端接口设计.md` 一致（请求体/响应体结构）
3. `docs/04` 中的所有接口均已在前端 `src/api/*.ts` 中对应封装
4. 响应体统一 `{ code, message, data }`，分页 data 为 `{ records, total, page, size }`
5. 需要登录的接口请求头携带 `Authorization: Bearer <token>`

## 7. 开发命令

```bash
cd frontend
npm install
npm run dev         # 开发（端口 5173，proxy /api → 8080）
npm run build       # 构建（dist/）
npm run preview     # 预览构建产物
```

## 8. 冷启动步骤

1. 确保环境：Node 22+、npm 10+、JDK 17、MySQL 5.7.26
2. 初始化数据库：`cmd /c "mysql -uroot -p123456 < backend\sql\init.sql"`
3. 进入 `backend`，执行 `mvn spring-boot:run`（端口 8080，context-path `/api`）
4. 进入 `frontend`，执行 `npm install && npm run dev`
5. 浏览器打开 `http://localhost:5173`

### 联调注意事项

- 后端 context-path 为 `/api`，前端 Vite 保留该前缀代理到后端
- 公开接口（GET 频道/标签/帖子/评论 等）无需登录
- 发帖/评论/点赞/收藏 等需要登录（`Authorization: Bearer <token>`）
- 管理后台（`/admin/*`）仅 `ADMIN` 角色可访问
