# 07 开发流程与协作

## 1. 团队角色
| 成员 | 职责 |
|------|------|
| A 组长 | 后端开发、接口文档、数据库、集成联调 |
| B | 前端开发 |
| C | 测试用例、Postman 集合、回归 |

## 2. 仓库与分支策略

### 仓库
- `school-forum`，推荐 GitHub / Gitee
- 三个目录：`backend/` `frontend/` `docs/`

### 分支模型
| 分支 | 用途 |
|------|------|
| main | 可发布稳定版本 |
| develop | 集成测试 |
| feature/* | 功能开发分支 |
| fix/* | 缺陷修复 |
| hotfix/* | 紧急修复 main |
| release/* | 发布候选 |

### 命名约定
- `feature/auth` `feature/post-crud` `feature/comment-tree` `feature/notification`
- `fix/login-401` `hotfix/sql-injection`

## 3. 提交规范
```
<type>: <subject>

<body>
```
- type: feat fix docs style refactor test chore
- 示例：`feat: 完成帖子 CRUD 接口`

## 4. 协作流程
1. 开发前从 develop 拉新分支
2. 每完成一个子功能推送
3. 提 Pull Request 到 develop，至少一人 Review
4. CI 通过后合并
5. 每周一次集成联调（develop 部署本地）

## 5. 接口先行约定
1. 后端先在 Knife4j 上挂接口 stub（仅签名 + 空实现返回 stub）
2. 前端按 stub 在 `api/*.ts` 写客户端 + Mock
3. 后端补完真实实现，前端切 baseURL 联调

## 6. 里程碑（4-5 周）

| 阶段 | 周次 | 产出 | 负责 |
|------|------|------|------|
| 设计 | W0 | 需求 / DB / 接口 / 骨架 | A 主导 |
| 基础 | W1 | 账户 + 频道 + 帖子主流程 | A/B 并行 |
| 互动 + 检索 | W2 | 评论 / 点赞 / 收藏 / 搜索 | A/B 并行；C 写接口用例 |
| 通知 + 后台 | W3 | 站内消息 / 举报审核 / 管理面板 | A/B 并行 |
| 推荐 + 优化 | W4 | 热门榜定时 / 前端完善 / 性能调优 | 全员 |
| 测试 + 文档 | W5 | 接口回归 / 用户手册 / 部署 / 答辩 | 全员 |

## 7. 沟通与文档
- 每周一次站会同步进度
- 接口变更同步更新 `docs/04-后端接口设计.md`
- 风险 / 阻塞记录在仓库 issues
